    SQL Injection 이란?

    SQL Injection은 SQL쿼리문에 악의적인 코드를 삽입하여 데이터베이스에 저장된 민감한 정보에 접근하거나 조작할 수 있는 취약점입니다.

    해당 취약점이 생기는 이유는 SQL쿼리문을 이용하는 데이터가 사용자가 입력한 데이터가 제대로 필터링되지 않을때 발생하게 됩니다.

    오픈소스 웹 애플리케이션 보안 프로젝트 OWASP 2021 TOP 10에 있는 Injection에 포함되어 3번째로 위험한 취약점입니다.

    가능한 공격

    데이터 탈취(Data Extraction)
    사이트 사용자의 민감한 개인정보 등 DB에 저장되어 있는 DB구조를 알아내고 데이터를 추출할 수 있습니다.

    데이터 변조(Data Modification)
    DB에 저장되어 있는 데이터를 수정하거나 삭제할 수 있습니다.

    인증 우회(Authentication Bypass)
    로그인 시 비밀번호를 무시하고 아이디만을 사용하여 인증을 우회 할 수 있습니다.

    SQL Shell 획득
    SQL Injection을 통해 데이터베이스에서 시스템 명령을 실행할 수 있는 SQL 쉘(shell)을 확보할 수 있습니다.

    파일 업로드 및 실행
    SQL Injection을 이용해 서버에 악성 파일을 업로드하거나 실행하여 추가적인 권한을 획득하거나 공격을 확장할 수 있습니다.

    거부 서비스 (DoS) 공격
    SQL Injection을 통해 복잡한 쿼리나 무한 루프 등을 실행하여 서버의 리소스를 소모하게 하여 서비스 거부 상태로 만들 수 있습니다.

    SQL Injection 간단한 예시

    예를 들어, 아래와 같이 작성된 PHP 코드가 있습니다.

    $sql = "SELECT id FROM user WHERE id='$_id' AND pass='$_pass' LIMIT 1";

    위 코드는 $_id나 $_pass의 값이 필터링을 거지치 않고 그대로 사용되기 때문에 SQL Injection 취약점이 생겼습니다.
    때문에 $_id 값으로 admin' or '1'='1을 입력하고 $_pass를 비워두거나 임의의 값을 입력하면, 아래와 같은 SQL문이 됩니다.

    $sql = "SELECT id FROM user WHERE id='admin' or '1'='1' AND pass='____' LIMIT 1";

    수정된 SQL문으로 실행을 하면 id가 'admin'인 사용자를 찾고 '1'='1'이 되어 항상 참이므로 'admin'이 식별만 되고 인증은 건너뛰어져서
    SELECT에 admin만 잡히기 때문에 admin으로 로그인 할 수 있게 됩니다.

    SQL Injection 취약점 보안 방법

    보안 방법로는 다음과 같은 방법이 있습니다

    • Prepared Statement
    • White List Filter

    Prepared Statement

    Prepared Statement 란 SQL쿼리를 실행할 때 SQL문만 미리 컴파일 해두고, ?자리에 데이터만 나중에 바인드 하는 방식을 말합니다.

    Prepared statement의 작동 원리는 다음과 같습니다.

    1. SQL 쿼리 준비: 변수 자리 표시자(플레이스홀더)를 사용하여 SQL 쿼리를 작성합니다.
      • 예: SELECT * FROM users WHERE username = ? AND password = ?.
    2. parse(구문 분석): 데이터베이스가 이 쿼리를 미리 파싱하여 최적화합니다.
    3. bind(치환): 바인드 시 실제 변수 값이 자리 표시자에 바인딩됩니다.
    4. execute(실행): 바인딩된 값으로 컴파일된 쿼리가 실행됩니다.
    5. fetch(인출): 검색된 데이터의 결과를 추출합니다.

    해당 방식으로 바인드 된 데이터는 쿼리 구조를 변경할 수 없기 때문에 SQL 인젝션 공격을 효율적으로 방어 할 수 있습니다.

    또한 기존 Statement 방식 과는 달리 SQL쿼리를 미리 파싱해서 나중에 데이터만 입력받아 수행하기 때문에 속도도 빠릅니다

    Prepared Statement 사용 시 주의

    주의사항으로 order by, column, table 부분에서는 Prepared Statement를 사용하지 못하기 때문에 White List 필터링 도 같이 사용하는 것이 중요합니다.

    White List Filter

    화이트 리스트 필터란 개발자가 원하는 값이 아닐경우 무조건 통과되지 않도록 설정 하는 것을 말합니다

    Prepared Statement가 사용되지 않는 order by, column, table 부분을 체크 하는 것을 권장합니다.

    모의해킹 시 주의할 점

    SQL Injection 취약점을 찾을 때에 제일 중요한 것은 DB를 최대한 건드리지 않고 취약점이 있다는 것만 확인 하는 것이 중요합니다

    insert, delete, update 금지!

    -> DB의 정보가 날라가는 사태 가 생길수 있기 때문에 왠만한 경우가 아니면 사용하지 않는 것이 좋습니다.
    -> 만약의 경우 때문에 insert, delete, update 사용하게 된다면or말고 and로만 사용하는 것이 좋습니다.

    주석 남용 금지!

    -> ctf나 대회는 괜찮지만 실제 서버에서는 대부분 sql문을 매듭짓는 것을 추천 합니다.

    데이터 변조 금지!

    -> 당연한 이야기이며 서버에 있는 다른 사람의 데이터를 임의로 뽑아 내거나 바꾸는 것은 절때 하지말아야 합니다.
