    Stored XSS란?

    사람들이 보게되는 곳(게시판 등)에 스크립트를 저장된 XSS를 말합니다.

    서버에 저장되기 때문에 사이트를 돌아다니다 저장된 페이지를 클릭하는 순간 XSS를 만나서 실행이 되는 위험한 취약점 입니다.

    Stored XSS의 원리

    예시를 들면 게시판 같은 곳에서 글 작성 하면 서버의 DB에 게시글에 대한 데이터가 저장된 뒤 저장된 게시글을 보여주기 위해 DB에서 데이터를 가져와서 HTML상에 보여주었을 때 해당 스크립트가 사용자의 브라우저에서 실행됩니다.

    다음은 예시 입니다.

    1. 글을 작성 할 때 알림창에 1이라는 내용이 뜰 수 있도록 <script>alert(1);</script>을 적은 글을 작성 합니다.

    2. 해당 스크립트는 서버측 게시판의 DB에 저장이 됩니다.

    3. 저장된 스크립트가 있는 게시글을 누군가 클릭했을 때 스크립트가 실행되게 됩니다.

    XSS 확인하는 방법

    XSS가 막혀있는지 확인하는 방법으로는 서버에서 사용자가 저장한 데이터를 불러올 때 <,',",>가 HTML Entity로 치환되어 있는지 확인하는 것입니다.

    위에 있는 글에서 작성되어있는 스크립트가 막혔을 때와 뚫려있는 경우를 간단한 그림 2개로 표현하겠습니다.
