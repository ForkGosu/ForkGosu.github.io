    XSS(Cross Site Scripting)란?

    스크립트를 실행 할 수 있는 코드를 다른 사용자의 브라우저에서 실행를 실행시키는 것입니다

    클라이언트에서 스크립트를 삽입해서 공격하는 기법을 크사 또는 크스스라고 불립니다

    오픈소스 웹 애플리케이션 보안 프로젝트 OWASP 2021 TOP 10에 있는 Injection에 포함되어 3번째로 위험한 취약점입니다.

    특징

    서버를 사용하는 사용자를 공격하는 것이 XSS의 특징입니다

    종류

    XSS의 종류는 크게 3가지로 나뉩니다
    Stored XSS : 서버에 저장될 수 있는 곳에 XSS를 삽입 (자세히 보기)
    Reflected XSS : URL에 GET 파라미터를 통해 XSS를 삽입 (자세히 보기)
    DOM based XSS : JS가 DOM 객체를 제어하는 곳에 XSS를 삽입 (자세히 보기)

    가능한 공격

    세션 하이재킹(Session Hijacking)
    주로 인증 및 로그인된 사용자 계정이 저장된 세션 ID를 무단으로 탈취하여 신원을 훔치거나 추가공격을 위해 사용됩니다.

    키 로거(Key Logger)
    웹 페이지에 숨어 있다가 사용자가 입력한 키 값을 공격자 컴퓨터로 전송받아 비밀번호 평문이나 민감한 개인정보를 가져가기위해 사용됩니다.

    마우스 스니퍼(Mouse Sniffer)
    키 로거(Key Logger)를 막기 위해 마우스를 통해 특정 번호를 입력하는 정보(마우스의 위치 x,y값)를 가져가기위해 사용됩니다.

    거짓 정보 추가
    사용자가 보여지는 페이지를 수정해서 보여줄 수 있고 피싱 사이트 처럼 만들고 사용할 수 있습니다.

    악성 스크립트 유포
    사용자에게 악의적인 목적으로 만들어진 모든 코드를 유포 할 수 있습니다.

    POC(Proof of Concept) 코드

    모의해킹 업무에서는 직접적인 공격을 하면 절때 안됩니다
    하지만 취약점을 이용한 공격이 가능함을 보여줘야하기 때문에 증명 코드 를 삽입해서 보여줍니다
    그래서 alert(), prompt(), confirm() 등 알림창만 띄우는 증명코드를 POC코드라고 합니다

    XSS 취약점을 빠르게 찾을 때 확인 사항

    조건 : <'"> 가 입력 되어야 합니다(HTML Entity 되는 곳은 빠르게 넘어가는 것을 추천)

    1. Stored
    • 글을 저장되기 전에 HTML Entity로 치환 되는지 확인(Request할 때 치환 되는지 확인)
    • 글이 저장된 후 HTML Entity로 치환 되는지 확인
    1. Reflected
    • 입력된 값(파라미터)이 HTML상에 표시되는지 확인(alert도 확인)
    • 파라미터의 방식을 POST -> GET 방식으로 변경 후 정상으로 파라미터가 받아지는지 확인
    • GET 파라미터가 HTML Entity로 치환 되는지 확인(Request할 때 치환 되는지 확인)

    XSS 막는 방법

    막는 방법로는 다음과 같은 방법이 있습니다

    • HTML Entity로 치환
    • HttpOnly 설정

    HTML Entity로 치환

    무작정 블랙 리스트 필터링으로 XSS에서 사용되는 <,',",>를 삭제 해버린다면 자신이 쓴 글자가 HTML상에서 사라져서 사용자들이 당황스러워 할 것입니다

    그래서 HTML Entity로 <,',",>에 해당하는 단어를 &it;,&gt;,&it;,&it;로 치환해 준다면 HTML상에 글자가 사라지지 않고 XSS를 예방 할 수 있습니다

    만약 HTML Editor를 사용중 이라면 복잡하지만 다음 단계를 따르면 XSS를 예방 할 수 있습니다

    1. 사용자가 입력한 파라미터에서 HTML 특수문자들을 전부 HTML Entity로 치환합니다
    2. 허용할 tag를 식별하고 다시 살립니다(화이트 리스트 기반)
      • img, a, h1, h2…
    3. 다시 살려준 tag 내에 악의적인 Event Handler들을 필터링 합니다(블랙 리스트 기반)
      • onerror…

    HttpOnly 설정

    HttpOnly는 document.cookie API에 접근을 차단하여 세션 및 쿠키 탈취를 방지하기 위한 설정입니다.

    하지만 해당 방법으로는 XSS에 대해 근본적으로는 막을 수 없기에 HTML Entity로 치환하는 방법이 가장 효과적입니다.
