    Burp Suite란?

    침투 테스트 및 취약점 찾기에서 사용되는 프록시 기반 도구 입니다.

    그래서 서버와 클라이언트 사이에서 데이터를 주고받을 때 Burp Suite를 통해서 주고 받아 데이터를 확인하거나 변조할 수 있게 만드는 프로그램입니다.

    예시 그림으로는 다음과 같습니다.

    Burp Suite 사용 전

    Burp Suite 사용 후

    Burp Suite 시작

    1. 먼저 Burp Suite 다운로드 사이트에 들어간 후 Go straight downloads를 클릭합니다.(https://portswigger.net/burp/communitydownload)

    2. 자신에게 맞는 운영체재를 확인하고 DOWNLOAD버튼을 클릭합니다.

    3. 설치가 된 후 실행하면 다음과 같이 프로젝트를 만드는 화면이 보이는데 임시로 써도 되기 때문에 임시로 메모리에 프로젝트를 저장한다고 한 뒤 Next를 눌러줍니다.

    4. 이후 프로젝트에서 사용할 설정을 선택하라고 나오는데 기본 설정 사용한다고 한 뒤 Start Burp를 눌러줍니다.

    Display 설정

    테마를 선택하고 한글 폰트가 깨지는 것을 방지하기위해 폰트를 바꿔줘야 합니다.

    1. Display를 설정을 하기위해 Setting에 들어갑니다.

    2. 다크 테마를 할지 밝은 테마를 할지 선택합니다.

    3. 한글이 포함된 글꼴로 수정 한 뒤, Character Sets을 UTF-8로 수정 합니다(한글 사용 시 폰트 깨짐 방지)

    Response 설정

    서버로 부터 받은 응답 값을 모두 확인하기 위해서는 다음과 같은 설정이 필요합니다.

    1. Request interception rules에서 맨 위 Edit클릭 후 (^gif$|^jpg$|^png$|^css$|^js$|^ico$|^svg$|^eot$|^woff$|^woff2$|^ttf$) 로 수정한 뒤, Response interception rules도 해당 사진과 같이 체크 삽니다

    Proxy 설정

    이제 서버와 연결되는 것이 아니라 Burp Suite로 데이터를 보내기 위해 Proxy 설정을 해줘야 합니다.

    하지만 저는 브라우저를 열 때는 Burp Suite에서 빠르게 시작 할 수 있게 제공되는 Proxy설정이 끝난 브라우저를 이용 할 것이기 때문에 해당 작업은 이런게 있다는 것을 보기만 해 두시면 될 것 같습니다.

    해당 설정 단계는 나중에 모바일이나 다른 프로그램 작업 시 Proxy 설정을 해야 할 때 필요한 설정이며 브라우저 사용 시에는 굳이 안해줘도 됩니다.

    Chrome 브라우저는 Proxy 설정을 하려면 Window 설정을 건드려야 하기 때문에 Firefox로 진행하겠습니다.

    1. Firefox의 설정을 엽니다

    2. 스크롤을 맨 밑으로 내려서

    3. 수동 프록시 설정으로 한 뒤 127.0.0.1:8080으로 설정 한 뒤 HTTPS에도 이 프록시를 사용 체크 후 확인을 누릅니다.

    4. Burp Suite에서도 127.0.0.1:8080이 되어 있는지 확인 합니다

    5. 이제 해당 firefox 브라우저로 접속하면 Burp Suite의 HTTP History에서 잘 Proxy되는 지 확인 할 수 있습니다.

    주 기능 설명

    많은 기능들이 있지만 일단 주로 사용하는 기능만 설명 드리겠습니다.

    • Open browser
    • HTTP history
    • Intercept
    • Repeater

    Open browser

    해당 버튼을 사용하면 Proxy 설정을 하지 않아도 Proxy되어 있는 브라우저를 제공받을 수 있습니다.(Chrome이라 Firefox보다 익숙한 것 같아서 저는 해당 기능을 자주 사용합니다.)

    HTTP history

    Proxy 탭에 있으며 HTTP 통신으로 주거나 받은 기록을 확인할 수 있는 로그창 입니다.

    해당 기능을 이용하여 데이터를 어떻게 보냈는지 또 무엇을 어떻게 받았는지 빠르고 자세히 분석 할 수 있습니다

    Intercept

    Proxy 탭에 있으며 통신을 주고 받을 때 가로채서 해당 Request나 Response를 수정해서 서버로 주거나 수정해서 브라우저에 보낼 수 있습니다.

    저는 받은 데이터를 변조하기 위해 주로 사용합니다.

    1. Intercept on 버튼을 클릭해서 가운데 그림에 빨간색 불빛이 되게 만듭니다.

    2. 만약 Request뿐만 아니라 Response도 멈추게 하고 싶다면 오른쪽 클릭 후 Do intercept에 Response to this request를 클릭 해줍니다.

    3. Forward를 클릭하면 서버로 보내지고 그 전에 수정할 사항이 있다면 수정 할 수 있습니다

    4. 만약 Do intercept에 Response to this request를 클릭 해줬다면 브라우저로 받을 데이터를 수정해서 보내 줄 수 있습니다.

    Repeater

    Repeater탭에 따로 있으며 Request를 수정해서 서버로 데이터를 건낼 때 여러 번 수정을 하며 빠르게 서버의 반응을 볼 수 있습니다.

    저는 데이터를 줄 때 변조하기 위해 주로 사용합니다.

    1. 여러번 수정 할 Request를 HTTP history에서 선택하여 오른쪽 클릭 후 Send to Repeater를 클릭하여 Repeater탭으로 보냅니다.

    2. Repeater탭으로 이동하여 왼쪽에 Get파라미터나 Post파라미터 등의 정보들을 수정 한 뒤 Send 버튼을 누르면 수정된 요청에 대한 결과를 받을 수 있습니다.

    3. 위의 사진에서 search_word라는 파라미터 값을 수정한 뒤 Send 버튼을 누르면 결과가 바뀌었다는 것을 알 수 있습니다.

    끝으로

    Burp Suite의 기능 중에 Decoder로 Base64를 디코드,인코드하는 기능이나 Comparer로 긴 글을 비교하는 유용한 기능도 있습니다.

    Intruder 기능으로 Brute Force(무작위 대입공격)이 가능 하지만 해당기능을 Community 버전에서 사용 시 요청이 느리며 제한적이라 직접 파이썬 코드로 구현해야 빠릅니다.

    지금 설명한 기능만으로도 분석만 잘 한다면 웹 해킹은 거의 다 가능 할 것이며 만약 다른 기능을 알고 싶다면 기능을 하나씩 검색해 보며 분석하는 것도 좋을 것 같습니다.

    블로그를 봐주셔서 감사합니다.
