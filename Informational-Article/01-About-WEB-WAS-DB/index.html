    웹 서버의 구조

    웹 서버에 대해 자세하게 알기 위해서는 먼저 WEB<->WAS<->DB 구조를 알아야 합니다.

    웹 서버(WEB Server)의 역활

    웹 브라우저(클라이언트)가 웹 서버로 요청을 보내면 웹 서버는 WAS로 받은 데이터를 전달 하고

    WAS에서 받은 정적인 콘텐츠(HTML, CSS, JavaScript)를 다시 웹 브라우저(클라이언트)로 전달 하는 역활을 수행 합니다.

    WAS(Web Application Server)의 역활

    클라이언트로부터 전달 받은 데이터를 사용해 프로그래밍 언어(PHP, JSP, ASP, etc…)를 실행하고

    프로그래밍된 로직에 따라 DB가 필요하면 DB도 사용해 정적인 콘텐츠를 만들고

    정적인 콘텐츠를 다시 웹 서버로 전달하는 역활을 수행 합니다.

    DB(DataBase)의 역활

    프로그래밍 언어가 실행될 때 만약 데이터의 저장이 필요하거나 데이터에 대한 조회가 필요할 경우

    데이터를 저장하거나 조회하는 역활 을 수행 합니다.

    저장하는 방식은 어떤 데이터 베이스(DataBase)를 쓰느냐에 따라 다릅니다

    데이터 베이스(DataBase)의 종류는 관계형(Relational), 키-값형(KV store), 객체형(Object), 문서형(Document), 컬럼 패밀리형(Column Family) 등이 있으며 주로 관계형(Relational) 데이터 베이스를 많이 사용 합니다

    MySQL 구조

    MySQL은 관계형 데이터 베이스로 데이터를 행(Row)과 열(Column)이 있는 표 형식으로 저장됩니다
    엑셀로 예를 들자면 다음과 같습니다

    명칭

    DataBase : 엑셀 파일
    Table : 엑셀의 시트
    row : 데이터 항목
    column : 데이터 값

    요약

    WEB : 사용자로 부터 요청 받은 데이터를 WAS에 전달 하고 WAS에서 처리 된 데이터를 다시 받아서 사용자에게 전달
    WAS : WEB으로 부터 요청 받은 데이터를 프로그래밍 언어로 실행한 후 다시 WEB Server로 전달
    DB : 프로그래밍 언어에서 데이터를 저장받거나 저장된 데이터를 조회한 후 프로그래밍 언어로 전달

    블로그를 봐주셔서 감사합니다.
