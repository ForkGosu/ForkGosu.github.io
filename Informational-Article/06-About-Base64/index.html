    Base64란?

    암호화를 복호화 할때 키가 필요한 것과는 다르게 Base64는 인코딩과 디코딩으로 문자의 모양만 바꿔둔 것 입니다.

    Base64는 64진법을 뜻하며 ASCII문자중 화면에 표시되는 문자들을 64진법으로 표현한 것을 뜻합니다.

    Base64 인코딩 원리

    변경하는 방식은 모든 문자를 6bit씩 자른 뒤 6bit에 해당하는 문자를 Base64의 표에서 찾아 치환하는 것입니다.

    문자열이 Man이라 했을때 인코딩 한다고 가정하면 해당 과정은 다음과 같습니다.

    1. Man문자를 8bit단위로 나타내면 다음과 같습니다.
      Man -> 01001101 01100001 01101110
    2. 8bit로 나타낸 문자를 6bit로 나눕니다.
      01001101 01100001 01101110 -> 010011 010110 000101 101110
    3. 6bit로 나눈 문자를 다시 Base64 표를 보고 변환시켜줍니다.
      010011 010110 000101 101110 -> TWFu

    만약 8bit형식을 6bit로 끊어내다가 2bit씩 남는다면 남은 2bit당 =를 추가하게 됩니다.

    1. Mana문자를 8bit단위로 나타내면 다음과 같습니다.
      Mana -> 01001101 01100001 01101110 01100001
    2. 8bit로 나타낸 문자를 6bit로 나눕니다.
      01001101 01100001 01101110 01100001 -> 010011 010110 000101 101110 011000 01
    3. 6bit로 나눈 문자를 다시 Base64 표를 보고 변환시켜줍니다.
      010011 010110 000101 101110 011000 01 -> TWFuYQ==

    이 처럼 2bit씩 남는다면 base64를 유추할 수 있게 만드는 ==는 이렇게 발생됩니다.

    디코딩은 이와 반대로 생각하시면 될 것 같습니다.

    Base64 사용 이유

    Base64로 인코딩하게 되면 전송해야될 데이터 양이 33% 더 늘어나게 되는데, 왜 Base64을 만들게 됬는지 궁금합니다.

    Base64를 사용하는 이유는 시스템간 데이터를 전달할 때 생기는 오류를 해결하고자 사용하였습니다.

    만약 binary데이터 중에 ASCII에 포함되지 않는 Data가 있다면 ASCII로 전달 시 Data손실이 발생 할 수 있기 떄문에 Base64를 이용해서 제어 문자와 특수문자를 제외한 안전한 출력 문자만 이용해 데이터만을 전달하여 Data손실을 막을 수 있었습니다.

    빠르게 사용하기

    Chrome브라우저는 Javascript를 실행하기위한 컴파일러인 V8엔진이 내장되어 있기 때문에 F12로 개발자콘솔을 사용하면 빠르게 사용이 가능합니다.
    atob("") : base64를 디코딩
    btoa("") : base64로 인코딩

    블로그를 봐주셔서 감사합니다.
