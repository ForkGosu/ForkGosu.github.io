(4) PHP 맛보기

 요약

요약

  1. 파라미터 : 사용자가 서버에 전달하는 데이터
  2. 파라미터 전달법 종류 2가지 (GET/POST)
  3. GET : 파라미터를 URL에 ?뒤에 붙혀서 전달
  4. POST : Request Body에 붙혀서 전달
  5. form태그를 이용해서 다른 method(페이지)로 파라미터 전달

목차

  1. PHP 파일 만들기
  2. 만든 파일로 실제 테스트

PHP 파일 만들기

  1. 이제 vscode를 열어서 php파일들을 만들어 준다

  2. login.php 파라미터 보낼 파일 내용

GET 방식<br>
<form method="GET" action="login_proc.php">
    <input type="text" name="id" placeholder="User ID"/>
    <input type="password" name="pass" placeholder="User Password"/>
    <input type="submit" value="login"/>
</form>

POST 방식<br>
<form method="POST" action="login_proc.php">
    <input type="text" name="id" placeholder="User ID"/>
    <input type="password" name="pass" placeholder="User Password"/>
    <input type="submit" value="login"/>
</form>
  1. login_proc.php, 파라미터 받을 파일 내용
GET 방식<br>
<?php echo $_GET['id']; ?><br>
<?php echo $_GET['pass']; ?><br>
<br>
POST 방식<br>
<?php echo $_POST['id']; ?><br>
<?php echo $_POST['pass']; ?><br>

GET : 파라미터를 URL에 ?뒤에 붙혀서 전달 POST : Request Body에 붙혀서 전달

form : 안에있는 input태그의 name을 파라미터로 전달 method : 파라미터 전달 방식(GET,POST) action : 파라미터를 받을 페이지 설정

만든 파일로 실제 테스트

  1. 내부아이피를 통해 login.php 파일로 접근한 뒤 GET방식에 ip,pass를 입력한 뒤 login클릭

  2. GET방식은 URL의 ‘?‘뒤에 정보가 그대로 노출되는 것을 볼 수 있다

  3. 내부아이피를 통해 login.php 파일로 접근한 뒤 POST방식에 ip,pass를 입력한 뒤 login클릭

  4. URL에 정보 노출이 되지 않는다! 파라미터는 Request Body에 저장되어있다고 함
