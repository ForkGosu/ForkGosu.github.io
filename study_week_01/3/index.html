(3) 동적 페이지

 요약

요약

  1. WEB <-> WAS <-> DB
  2. WEB : 전달밖에 모르는 바보
  3. WAS : WEB이 PHP같은 동적 페이지는 WAS로 넘겨서 처리후에 클라이언트로 전달
  4. 요약 : WAS가 있는 도커를 실행 후 VScode-SFTP 연결해서 삶의 질을 올리자 - VScode_SFTP_세팅_바로가기

목차

  1. WAS가 있는 도커 실행
  2. 백엔드 코드란?

WAS가 있는 도커 실행

$ cd ~/webDev
-> 홈 디렉토리에 있는 webDev폴더로 이동

$ sudo docker ps -a  
-> 도커 실행중 인지 확인

$ sudo ./dockerCMD & 
-> WAS가 있는 도커 백그라운드로 실행

실행이 된다면 해당화면과 같이 나오고 멈춘것 같다면 엔터를 누르면 됨

이제 허접하게 Web서버만 있지않고 WAS가 있어서 PHP처리 가능

이제 “~/webDev/webApp” 경로는 웹서버 루트로서 폴더에 파일을 업로드하면 웹서버가 알아서 처리

VScode - sftp 세팅하면 삶의 질이 달라진다고 함(그래서 설정해봄)
설정방법 : VScode_SFTP_세팅_바로가기

백엔드 코드란?

백엔드 코드는 클라이언트에서 안보임
WAS가 서버에서 처리하고 나서 클라이언트로 전달했기 때문

PHP(백엔드 언어)
$_GET[’name’] : GET Method로 전달받은 파라미터 파라미터 : 사용자가 웹서버에게 전달하는 데이터

GET : 파라미터를 URL에 ?뒤에 붙혀서 전달

Front-End VS Back-End
Front-End : 브라우저가 실행하는 코드(HTML, CSS, Javascript…)
Back-End : 서버가 실행하는 코드 (PHP, ASP, JSP…)
