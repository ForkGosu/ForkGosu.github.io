(2) 웹 서버의 역활

 요약

요약

  1. 웹서버 : 딸깍(전달)
  2. 웹브라우저 : 그려야한다
  3. 요약 : 파이썬으로 웹서버 실행, URL(프로토콜,아이피,포트번호,파일경로)에 대해

목차

  1. 파이썬 웹서버 실행
  2. URL에 대해

파이썬 웹서버 실행

$ cd webBasic/
-> webBasic이라는 폴더로 이동

$ sudo python3 -m http.server 80
-> 파이썬을 사용해서 80번 포트를 http서버로 열기
-> sudo는 비밀번호를 입력해야함 (student1234)

잘 실행 된다면 해당 화면이 보인다


URL에 대해

URL경로
[Protocol]://[Domain or IP]:[Port]/[File Path]
-> http://[내부IP]/[파일위치]
-> http://192.168.12.128/test.txt

URL을 이용해서 브라우저로 들어가보자


파일위치에서는 ..같은 상위 폴더로 돌아가는 경로는 URL에 안먹힘
-> 루트경로까지 돌아갈수 있다면 그건 서버입장에서 자살행위

캐시 : 한번 실행했던 브라우저 정보는 클라이언트가 저장해서 사용하여 자원 아끼려함
-> 때문에 서버에서 업데이트가 일어나면 정확한 정보를 위해 캐시 삭제 필요

웹서버 : 그냥 데이터만줌
웹브라우저 : 데이트를 참고해서 이쁘게 그리는 녀석 착한녀석

이런 느낌일까?

웰논포트 : 서버통신에서의 사회적 약속
다른 포트 사용해도 되지만 대부분 사람들은 이렇게 사용한다
http -> 80
https -> 443
ssh -> 22

python으로 webServer포트번호 다르게 사용해보기

$ sudo python3 -m http.server 9999
-> 9999번 포트로 http서버로 열면 더이상 포트생략 불가

-> http://192.168.12.128/test.txt 이렇게 80포트 생략해서 URL입력했던 걸
-> http://192.168.12.128:9999/test.txt 이렇게 9999포트 명시해서 URL입력 해야 함

NAT
-> 왜 인터넷망에있는 서버를 빌려야 하는지 답이 나올것이다

기본적으로 index파일을 전달(이것도 약속)
